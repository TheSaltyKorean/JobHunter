Job Types & Resumes
Define your resume types (1–6). Each type has a name, emoji, keywords for auto-detection, and a PDF resume file. The first type is the default when no keywords match.
Resume files: Place PDF files in the extension's resumes/ folder using the filename you set for each type (e.g. cloud.pdf).

Auto-detection: When you open a job listing, the title and company are matched against each type's keywords. The first match wins. If nothing matches, the first type is used as default.

Domain / URL Overrides
How matching works: When Auto-Fill encounters a login page, it checks for a URL-specific override first (e.g. microsoft.myworkdayjobs.com), then a domain override (e.g. myworkdayjobs.com), then falls back to your default credentials.

Security note: All credentials are stored locally in Chrome's extension storage (never synced or sent anywhere).

When Auto-Fill encounters a question it can't match, Claude CLI will analyze the question and generate an appropriate answer based on your profile. Requires the companion server to be running.
Not checked
How to start: Run the companion server from your extension folder:

The server wraps your local claude CLI — no API key needed, uses your existing Claude subscription. Without the server running, unrecognized fields are simply skipped.
